任务：
阅读《投资学精要（第九版）》第3章《证券市场》的全部内容，基于书中文字、表格、公式、案例与图片，写一份详细、可靠、适合自学的中文 tutorial 风格研究报告，并直接产出可编译 LaTeX 与渲染后的 PDF。

优先使用以下材料：
- 正文：`mineru/output/投资学精要.full.md`
- 备用切分稿：`mineru/output/投资学精要.part1-600/full.md`、`mineru/output/投资学精要.part601-764/full.md`
- 图片与图表：`mineru/output/**/images/*`

材料使用约束：
- 上述 MinerU Markdown 只用于阅读、定位章节边界、核对 OCR、提取图表线索与建立证据台账。
- 禁止把这些 Markdown 内容直接充当最终报告正文，或先写成章节 Markdown 再转写/导入到 LaTeX；辅助性质的 `source-ledger.md`、工作笔记或检查记录可以单独保存。
- 禁止把这些 Markdown 通过 `markdown` 宏包、`\markdownInput`、`\input{*.md}`、Pandoc 中转，或任何“先写 Markdown 再塞进 LaTeX”的方式并入最终报告。

执行要求：
1. 必须完整覆盖本章正文，不要只看目录、学习目标或章末习题。
2. 先在 MinerU 结果中定位本章起止范围，再阅读本章相关图片、图表、公式和案例。
3. 必须完整遵循 `$deepresearch-skill` 的原生 LaTeX/PDF 工作流，直接写原生 LaTeX 报告，不要退化成 Markdown 中间层流程。
4. 在忠于教材结构和逻辑的前提下重写，不要大段照抄原文。
5. 可以进行适度联网研究，用来补充概念来源、现代市场实践、近年制度变化、经典论文或权威机构资料，但要明确区分“教材内容”和“外部补充”。
6. 对书中已经过时的数据、制度、案例或技术，补充说明其时效性以及与今天的差异。
7. 如果 OCR 有疑似错误，请结合上下文、原图或原页进行修正；拿不准时明确标注不确定。
8. 解释公式时给出变量含义、经济直觉、适用条件，并尽量配一个简短例子。
9. 对图片和图表不要略过，要说明图里展示了什么、为何重要、支撑了哪一个概念。图片要在文章中，而不是堆放在开头或者结尾。
10. 面向中文读者写作，术语首次出现时尽量给出中英文对照。
11. 如果需要 `source-ledger.md`、`sources.md`、方法记录、OCR 勘误笔记或工作清单，可以另存为辅助材料；但它们不能替代最终报告正文，也不能被直接 `\input`、`\markdownInput` 或其他 Markdown 管线并入最终 LaTeX。
12. 最终正文、表格、列表、脚注、引用说明、附录都必须是原生 LaTeX 表达。
13. 用 nord forest theme，中文字体用 LXGW。
14. 不要访问 tutorial-backup
15. 对于图片，优先使用 minerU 的图片，因为我们自己的裁切不是很准确。

产出要求：
- 所有输出必须放到 `tutorial/ch03/` 目录下，不要写到其他目录。
- 主 LaTeX 文档保存为 `tutorial/ch03/report.tex`。
- 渲染后的 PDF 保存为 `tutorial/ch03/report.pdf`。
- 如果需要拆分正文、附录、来源台账、封面或方法说明，只能拆成 `tutorial/ch03/` 内的 `.tex` 子文件。
- 图片、图表、截图等资产统一放在 `tutorial/ch03/figures/` 或 `tutorial/ch03/assets/`。
- 允许保存 `source-ledger.md`、`sources.md`、方法记录等辅助 Markdown sidecar；但禁止把章节主教程正文另存为 `tutorial.md` 或其他 Markdown 正文后再转写/导入到 LaTeX。

- 对联网补充内容给出来源链接，并在报告中保持来源可追溯。

写作风格：
- 详细、清晰、可教学，不要写成堆砌名词的综述。
- 优先解释“为什么”，再解释“怎么算”和“怎么用”。
- 如果某一节特别抽象，给一个小例子或简单类比帮助理解。
