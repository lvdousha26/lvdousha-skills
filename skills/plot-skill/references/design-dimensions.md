# 图表设计维度

这份文档回答一个核心问题：画一张图时，到底有哪些方面要考虑。

## 1. 信息任务

先回答这张图在表达什么：

- 比较：谁高谁低
- 趋势：怎么随时间或 step 变化
- 相关性：两个变量怎么一起变化
- 组成：整体由哪些部分构成
- 多维画像：一个对象在多个指标上的 profile

图类型必须先服从信息任务，再谈风格。

## 2. 图形几何

也就是用什么 mark 来承载数据：

- `bar`：长度
- `line`：位置 + 连续趋势
- `scatter`：二维位置
- `donut`：角度 / 面积
- `radar`：多轴轮廓

几何层是图的骨架，配色只是覆盖层。

## 3. 坐标与尺度

- 是否必须从 0 开始
- 是否用对数轴
- 是否做归一化
- 是否离散刻度
- 是否需要 broken axis / inset

这一步直接影响图是否诚实。

## 4. 视觉编码

- 位置
- 长度
- 颜色
- 形状
- 透明度
- 线宽
- 填充纹理

一般优先级是：

`位置 > 长度 > 颜色 > 形状 > 透明度`

也就是说，不要让颜色去承担本该由坐标或几何承担的信息。

## 5. 风格层

风格层关注的是统一语言，而不是单张图的内容：

- 配色系统
- marker 系统
- 线型系统
- 边框风格
- 网格风格
- baseline 风格
- 强调策略

这正是 preset 应该承载的内容。

## 6. 标注层

- 标题
- 轴标签
- legend
- 数值标签
- 注释框
- 参考线说明

大多数图“乱”，不是因为颜色乱，而是标注策略没收住。

## 7. 版式层

- 长宽比
- 单图还是多子图
- 留白
- legend 放图内还是图外
- 标题和图的距离

这一步决定图是否挤、是否像论文图。

## 8. 输出层

- 论文图
- 报告图
- slides 图
- 网页图

不同输出场景，对比度、信息密度、线宽、标注密度都不同。

## 对 skill 的启发

后续维护 preset 时，不要按“某篇论文名”组织，而要按这些维度组织。
