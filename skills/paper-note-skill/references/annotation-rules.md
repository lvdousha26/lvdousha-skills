# 批注规则

## 核心判断流程

每条批注统一按下面四步判断：

1. 这句话在整篇文章里承担什么功能。
2. 它是否完成了这个功能。
3. 如果没有完成，主要问题属于哪一类：信息不足、逻辑跳步、证据不够、位置不当，还是表述不稳。
4. 这一处更适合保留、补强、重写，还是重排。

说明：

- 功能决定颜色和功能标签。
- 判断决定批注内容。
- `保留 / 补强 / 重写 / 重排` 只作为判断标签使用，不通过颜色表达。

## 标注载体与颜色规则

全文只使用四种标注手段：

- 句子颜色：直接标在正文句子上，只表示这句话在做什么
- 重点色标记：用于少量特别关键的 insight、结果句或最该记住的句子
- `changes`：用于正文句子着色
- `tcolorbox`：用于节末总评

如果启用 `\pnote` 分析，英文原句块后允许再接：

- `\pnote{功能|判断}{评}`

中文对应句子如果保留翻译，也应使用与英文原句相同的功能色，不要一边着色、一边退回纯灰说明句。
中文对应句子默认 inline 接在英文后面，不单独起段。

颜色只表示“这句话在做什么”，不表示好坏。

| 颜色 | 功能 |
| ---- | ---- |
| 蓝色 | 背景 / 问题定义 |
| 紫色 | 空缺 / 限制 |
| 洋红 | 研究问题 / 核心切入点 |
| 绿色 | 方法 / 贡献 |
| 橙色 | 结果 / 证据 |
| 红色 | 主张 / 外推 |
| 灰色 | 结构 / 过渡 |
| 青色 | 文献定位 / related work |
| 金色 | 重点句 / insight 标记 |

使用原则：

- 颜色只标功能，不标好坏。
- 金色尽量作为叠加标记使用，少用。
- 如果一句话只是定义术语，优先并入“方法”。
- 如果一句话只是交代实验设置，优先并入“结构”或“结果”。

## 批注粒度

| 章节 | 批注粒度 | 主要检查点 |
| ---- | ---- | ---- |
| 摘要 | 按相邻同功能块细批 | 是否按“问题 -> 空缺 -> 方法 -> 结果 -> 主张”推进 |
| 引言 | 按相邻同功能块细批 | 是否完成 `Move 1 / Move 2 / Move 3` |
| 相关工作 | 按相邻同功能块细批 | 是否建立前人工作与本文关系，而不是罗列 |
| 方法 | 只批关键定义、关键公式、关键模块说明 | 定义是否及时，模块是否回应前文 gap |
| 实验与消融 | 只批关键段落 | 设置是否支撑 claim，消融是否验证组件作用 |
| 分析与结论 | 只批影响主线判断的句子 | 是否真正收束主线，是否越证据外推 |

补充说明：

- 句子颜色仍然逐句标，但 `\pnote` 默认按相邻、同功能、同一局部推进块来合并。
- 摘要和引言默认每个完整句子都要有功能判断，但不要求每句都单独挂一个 `\pnote`。
- 只要相邻几句承担的是同一种功能，就优先合并成一个 `\pnote`，把注释放在该功能块最后一句中文后面。
- 如果一句话单独承担了明显不同的功能，或发生了清楚的推进转折，就不要并到前后句里一起批。
- caption 默认不翻译，也不做逐句分析。
- references、参考文献页、BibTeX 数据、`thebibliography` 环境和纯引用条目不做逐句分析。

## `\pnote` 格式

如果启用功能块分析，固定格式如下：

```latex
\pnote{功能|判断}{评}
```

常用功能标签：

- 背景
- 问题定义
- 空缺
- 研究问题
- 方法
- 结果
- 主张
- 结构
- 文献定位

常用判断标签：

- 保留
- 补强
- 重写
- 重排

补充规则：

- 不再在 `\pnote` 里写逐句翻译。
- 默认不写定位链接、编号或引句。
- 只有脱离上下文后批注意图不清，才补最短引句。
- `评` 里不要直接写裸 LaTeX 数学宏；必须写数学记号时，统一包进 `$...$`。
- 如果正文原句还要带 `\footnote` 或 `\footnotemark`，不要把它包进 `\bgsent`、`\methodsent`、`\resultsent` 等句子功能宏里；应先结束句子功能宏，再把脚注命令接在外面。
